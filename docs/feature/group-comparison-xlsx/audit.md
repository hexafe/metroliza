# Group Comparison XLSX audit

## Metadata
- **Owner:** Data Export & Analysis Team
- **Status:** Active (draft)
- **Scope:** Audit current workbook export behavior and identify the gaps required to support a dedicated Group Comparison XLSX flow.
- **Exit criteria:** Findings are either resolved in implementation work or converted into tracked follow-up items with clear owners.

## Current behavior
- Group comparison output is currently documented at a high level from the user perspective, but this folder does not yet have a dedicated implementation workspace for XLSX-specific planning.
- The active docs index points users to the root `README.md` for Group Comparison worksheet interpretation and statistical caveats, which suggests the feature is already conceptually exposed but not yet tracked here as a standalone workstream.
- Export-related module boundary notes in `docs/README.md` indicate that workbook orchestration, chart payload shaping, and aggregation logic are already split across helper modules, which is likely the main implementation surface for any Group Comparison XLSX work.

## Confirmed bugs
- No code-path audit has been attached to this doc set yet, so there are currently no confirmed implementation bugs captured here.
- A documentation gap is confirmed: there is no dedicated active feature-doc workspace for Group Comparison XLSX planning, ownership, and validation tracking.
- Root-cause confirmation for any stats, worksheet layout, or chart defects is still pending code and workbook inspection.

## UX findings
- Users currently have to infer Group Comparison export details from broader documentation instead of a focused feature workspace, which makes planning and acceptance criteria harder to review.
- A worksheet-oriented deliverable should prioritize obvious sheet naming, frozen headers, readable summary placement, and low-friction navigation between raw values, summaries, and charts.
- Any workbook generated for non-technical users should avoid forcing interpretation of statistical caveats solely from chart labels; the worksheet itself should carry enough context.

## Chart findings
- Current docs imply chart generation is part of the export pipeline, but this workspace does not yet capture chart-specific acceptance criteria.
- Group comparison charts will need explicit readability checks for category labeling, legend clarity, axis scaling, and color differentiation between groups.
- If statistical summaries and charts are both present in the workbook, chart captions or nearby notes should make clear whether values represent counts, percentages, means, confidence intervals, or other derived metrics.

## Test coverage findings
- There is no feature-specific test checklist yet for Group Comparison XLSX.
- Existing docs highlight helper-module boundaries, which is a good indicator that test coverage should be split between pure aggregation/statistics helpers, workbook layout generation, and chart payload construction.
- Workbook snapshot or structural assertions will likely be needed in addition to unit tests so regressions in sheet layout, formulas, and chart anchoring are detectable.

## Open questions
- What exact statistical outputs are required in the first release: counts only, descriptive statistics, pairwise comparisons, p-values, confidence intervals, or effect sizes?
- Is Group Comparison XLSX a new export mode, an extension of an existing workbook, or a renamed version of current Group Comparison output?
- Which user-facing worksheet tabs are mandatory in v1, and which are optional or deferred?
- Should charts be generated by default, behind a toggle, or only when the data shape meets readability thresholds?
- What is the minimum acceptable automated test strategy for workbook/chart validation in CI?
